Containerized Personal Portfolio with Automated CI/CD

Problem: Even simple websites are often deployed manually, without versioned images or repeatable deployments.

Approach: Built a fully containerized portfolio using Docker + Nginx (alpine), with a clean Dockerfile and .dockerignore. Configured GitHub Actions to build an image on every push to main, tag it as latest and with the short commit SHA, and push it to GitHub Container Registry (GHCR).

Outcome: Treats the portfolio as a real DevOps project: every commit produces a versioned, registry-hosted image that can be pulled and run anywhere with docker run. This demonstrates end-to-end CI (build, tag, push) and container-based delivery.

How I Build Projects

Whether it‚Äôs a small Bash script or a full AI platform, my approach stays consistent:

1. Start from a real problem

I focus on what is actually slow, confusing or unreliable for users or teams, and turn that into a clear problem statement.

2. Design before code

I sketch flows, failure points, data paths and security concerns so the solution is structured, not just a quick script.

3. Automate and observe

I automate repetitive steps (scripts, Docker, CI/CD) and add logs/visibility so issues are easy to detect and debug later.

4. Iterate, document, share

I refine based on real usage, document the setup, and keep things in Git so others can run, reuse, or extend the work easily.
